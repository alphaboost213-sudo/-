# ImgUniq — уникализатор изображений

Сервис принимает ссылку на изображение и выдаёт статическую ссылку вида:

```text
https://твой-сервис.railway.app/img/abc123
```

При каждом открытии этой ссылки сервис заново скачивает исходник, применяет случайные микротрансформации и отдаёт новый JPEG-файл.

## Что теперь делает уникализация

При каждом рендере применяется набор лёгких случайных изменений:

- случайный шум;
- низкоамплитудная текстура;
- микросдвиг яркости, контраста и насыщенности;
- лёгкий blur или sharpening;
- микрокроп 0–2 px с ресайзом обратно;
- случайный ресемплинг через промежуточный размер;
- субпиксельный affine-сдвиг;
- JPEG re-encode с плавающим `quality`;
- случайные параметры JPEG `progressive` и `subsampling`;
- новый EXIF-блок при каждом сохранении.

Визуально изображение почти не меняется, но технически файл каждый раз другой.

## Хранение ссылок

Ссылки больше не хранятся в RAM. Используется SQLite:

```text
imguniq.sqlite3
```

Для Railway лучше подключить Volume и задать переменную окружения:

```text
DATA_DIR=/data
```

Тогда база будет лежать здесь:

```text
/data/imguniq.sqlite3
```

Без Volume база может пропасть при пересборке или переносе окружения Railway.

## Как деплоить на Railway

### Через GitHub

1. Создай репозиторий на GitHub.
2. Загрузи туда файлы проекта.
3. Railway → New Project → Deploy from GitHub repo.
4. Подключи Volume.
5. Mount path: `/data`.
6. Variables → добавь `DATA_DIR=/data`.
7. Deploy.

### Через Railway CLI

```bash
npm install -g @railway/cli
railway login
railway init
railway up
```

## API

### Создать одну ссылку

```http
POST /api/register
Content-Type: application/json

{"url":"https://example.com/image.jpg"}
```

### Создать пачку ссылок

```http
POST /api/register-batch
Content-Type: application/json

{"urls":["https://example.com/1.jpg", "https://example.com/2.jpg"]}
```

Лимит пачки — 50 ссылок за раз.

### Получить статистику

```http
GET /api/stats
```

## Важно

- SQLite хранит только исходные ссылки, не сами изображения.
- При каждом открытии `/img/<id>` исходная картинка скачивается заново.
- Если исходная ссылка перестанет работать, уникализированная ссылка тоже перестанет отдавать изображение.
- Для полного автономного хранения картинок нужно отдельно добавлять скачивание оригиналов в `/data/uploads`.
