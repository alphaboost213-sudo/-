# ImgUniq — обработка изображений и видео с авторизацией

Сервис принимает ссылку на изображение или видео и выдаёт статическую ссылку вида:

```text
https://твой-сервис.railway.app/img/abc123
https://твой-сервис.railway.app/video/def456
```

При каждом открытии этой ссылки сервис заново скачивает исходник, применяет лёгкие случайные микротрансформации и отдаёт новый файл.

## Что добавлено в этой версии

- вход по логину и паролю;
- доступ к панели только для добавленных пользователей;
- самостоятельной публичной регистрации нет;
- админ-страница `/admin` для добавления известных пользователей;
- список пользователей с логинами и заданными паролями, чтобы админ мог посмотреть пароль, если человек забудет;
- логи входов: успешный вход, неправильный пароль, IP, время;
- логи действий: кто создавал ссылки;
- для видео создаётся до `5` вариантов на одну исходную ссылку;
- лимит длины видео — `90 секунд` по умолчанию.

## Авторизация

После деплоя главная страница `/` требует вход.

Первый админ создаётся автоматически:

```text
login: admin
password: берётся из ADMIN_PASSWORD или генерируется автоматически
```

Лучший вариант — сразу добавить в Railway переменные:

```text
ADMIN_LOGIN=admin
ADMIN_PASSWORD=твой_пароль_админа
SECRET_KEY=любая_длинная_случайная_строка
```

Если `ADMIN_PASSWORD` не задан, сервис сам сгенерирует пароль и сохранит его в:

```text
/data/admin_password.txt
/data/secret_key.txt
```

Также при первом запуске он напишет пароль в Railway logs:

```text
ImgUniq admin login: admin
ImgUniq admin password: ...
```

После входа админ может открыть:

```text
/admin
```

Там можно добавить пользователей. Смена пароля пользователем не предусмотрена.

Если логин или пароль неверный, на странице входа будет написано:

```text
Неправильный пароль
```

Важно: пароли, которые задаёт админ, показываются в админке. Введённые неправильные пароли в логи не пишутся — пишется только факт ошибки, логин, IP и время.

## Публичность готовых ссылок

По умолчанию `/img/<id>` и `/video/<id>` остаются публичными, чтобы их могли открывать внешние рендереры и динамические ссылки.

```text
PUBLIC_MEDIA_LINKS=1
```

Если поставить:

```text
PUBLIC_MEDIA_LINKS=0
```

то даже готовые ссылки `/img/...` и `/video/...` будут требовать вход. Но внешние сервисы без cookie авторизации тогда не смогут открыть эти ссылки.

## Что делает обработка изображений

При каждом рендере изображения применяется набор лёгких случайных изменений:

- случайный шум;
- низкоамплитудная текстура;
- микросдвиг яркости, контраста и насыщенности;
- лёгкий blur или sharpening;
- микрокроп 0–2 px с ресайзом обратно;
- случайный ресемплинг через промежуточный размер;
- субпиксельный affine-сдвиг;
- JPEG re-encode с плавающим `quality`;
- случайные параметры JPEG `progressive` и `subsampling`;
- новый EXIF-блок при каждом сохранении.

## Что делает обработка видео

Видео обрабатывается через `ffmpeg`:

- исходное видео скачивается во временный файл, не в RAM;
- длина видео проверяется, максимум по умолчанию `90 секунд`;
- каждый запрос перекодируется в MP4/H.264;
- создаётся до `5` отдельных вариантов ссылок на одно видео;
- у каждого варианта немного отличается диапазон фильтров;
- применяется лёгкая коррекция яркости, контраста и насыщенности;
- применяется слабый hue-сдвиг;
- добавляется очень слабый шум;
- иногда применяется мягкий sharpening;
- плавает `CRF` и битрейт аудио;
- старые метаданные очищаются и записываются новые технические метаданные.

По умолчанию лимит видео — `80 MB`, таймаут обработки — `180 секунд`, длина — `90 секунд`, вариантов — `5`.

Можно изменить переменными окружения:

```text
MAX_VIDEO_BYTES=83886080
VIDEO_TIMEOUT=180
VIDEO_MAX_SECONDS=90
VIDEO_VARIANTS_DEFAULT=5
VIDEO_VARIANTS_MAX=5
```

## Хранение

Используется SQLite:

```text
imguniq.sqlite3
```

Для Railway лучше подключить Volume и задать переменную окружения:

```text
DATA_DIR=/data
```

Тогда база и автоматически созданный пароль админа будут храниться в `/data`:

```text
/data/imguniq.sqlite3
/data/admin_password.txt
/data/secret_key.txt
```

Без Volume база, пароль и secret key могут пропасть при пересборке или переносе окружения Railway.

## Как деплоить на Railway

### Через GitHub

1. Создай репозиторий на GitHub.
2. Загрузи туда файлы проекта.
3. Railway → New Project → Deploy from GitHub repo.
4. Подключи Volume.
5. Mount path: `/data`.
6. Variables → добавь:

```text
DATA_DIR=/data
ADMIN_LOGIN=admin
ADMIN_PASSWORD=твой_пароль_админа
SECRET_KEY=любая_длинная_случайная_строка
```

7. При необходимости добавь:

```text
MAX_VIDEO_BYTES=83886080
VIDEO_TIMEOUT=180
VIDEO_MAX_SECONDS=90
VIDEO_VARIANTS_DEFAULT=5
VIDEO_VARIANTS_MAX=5
PUBLIC_MEDIA_LINKS=1
```

8. Сделай Deploy или Redeploy.

`ffmpeg` подтягивается через Python-зависимость `imageio-ffmpeg`, поэтому отдельный системный пакет обычно не нужен.

## API

Все API для создания ссылок требуют авторизацию через браузерную сессию.

### Создать ссылку на изображение

```http
POST /api/register
Content-Type: application/json

{"url":"https://example.com/image.jpg", "type":"image"}
```

### Создать 5 вариантов видео

```http
POST /api/register
Content-Type: application/json

{"url":"https://example.com/video.mp4", "type":"video", "variants":5}
```

Ответ содержит `results` со всеми вариантами:

```json
{
  "success": true,
  "results": [
    {"variant": 1, "unique_url": "https://.../video/abc"},
    {"variant": 2, "unique_url": "https://.../video/def"}
  ]
}
```

### Создать пачку ссылок

```http
POST /api/register-batch
Content-Type: application/json

{"urls":["https://example.com/1.jpg", "https://example.com/2.jpg"], "type":"image"}
```

Для видео:

```http
POST /api/register-batch
Content-Type: application/json

{"urls":["https://example.com/1.mp4"], "type":"video", "variants":5}
```

Лимит пачки: `50` ссылок для изображений и `5` исходных видео за раз. Для каждого исходного видео можно создать до `5` вариантов.

### Получить статистику

```http
GET /api/stats
```

## Важно

- SQLite хранит исходные ссылки, пользователей, заданные админом пароли и логи входов/действий.
- При каждом открытии `/img/<id>` исходная картинка скачивается заново.
- При каждом открытии `/video/<id>` исходное видео скачивается и перекодируется заново.
- Если исходная ссылка перестанет работать, созданная ссылка тоже перестанет отдавать файл.
- Видео тяжелее изображений: для очень активного использования лучше позже добавить очередь задач и хранение готовых файлов в `/data` или S3/R2.


## Важно про пароль админа

В этой версии пароль администратора каждый раз синхронизируется из переменной Railway `ADMIN_PASSWORD`.
Если админ был создан раньше со случайным паролем, просто поменяйте `ADMIN_PASSWORD` в Railway и сделайте Redeploy — логин `admin` начнёт принимать новый пароль.
